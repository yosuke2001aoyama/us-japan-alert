# JPUS Alert — 米国・日米関係速報

日米の主要報道、政府・議会の一次情報、要人発信を横断し、日本政府の局長・課長級が把握すべき政策案件を抽出する公開ダッシュボードです。

## できること

- White House、国務省、国防総省、Congress.gov、首相官邸・外務省・防衛省などの一次情報を選別取得
- Truth Socialの公開APIから @realDonaldTrump の発信を取得
- Google News RSSを使い、日米の主要紙・通信・テレビ・政治メディアを横断
- 首脳・閣僚、同盟・安保、通商・制裁、議会・重要人事を優先し、領事・在留・定例広報を除外
- 訪米や首脳会談の調整・検討・観測段階の報道も監視
- 画面は2分ごとに自動更新。時系列順／重要度順、対象期間、分野、一次情報で絞り込み
- 英語記事は「仮訳」を押したときだけ、見出し・要旨の機械翻訳を表示
- 同一URLの重複排除、検索、原文ハイパーリンク、CSV出力
- GitHub Actionsで5分ごとにスナップショットを保存し、重要新着をGitHub Issueで通知
- 一つの取得元が落ちても、他の監視を継続

## ローカル起動

```bash
npm ci
npm run dev
```

## 通知

標準では、GitHub Actionsに自動付与される権限だけで重要新着のIssueを作成し、リポジトリ所有者を担当者に設定します。GitHubの通知設定を通じてメールでも受け取れるため、Secretsは不要です。

Resendから直接メールを送りたい場合だけ、Actions secrets に次を登録します。

- `RESEND_API_KEY`
- `ALERT_TO_EMAIL`
- `ALERT_FROM_EMAIL`（Resendで認証済みドメインの送信元）

Actions variable `PUBLIC_DASHBOARD_URL` に公開URLを登録します。メール用の秘密情報が未登録でも、収集・公開画面・GitHub通知は動きます。

## 運用上の注意

「全メディア」の完全取得を保証するものではありません。公開RSS、検索RSS、公開APIで取得可能な範囲を広く監視し、ペイウォール記事は見出しと原文リンクを扱います。スクレイピング回避、出典明示、原文確認を前提としています。各提供元の仕様変更や利用規約変更に応じて取得経路を更新してください。
